# Test Report

Replace this file with your project's latest test state.

---

## How to Use This File

`TEST_REPORT.md` is the single source of truth for the most recent test verification state.

Claude reads this file during preflight to understand whether pre-existing failures exist.

Update this file after every test run.

---

## Template

```markdown
## Latest Run

- Feature: [Stage ID] — [Feature Name]
- Status: All tests passing / Failures present
- Date: [Date]

## Verification Commands

### Format
[command]
Result: PASS / FAIL

### Static Analysis
[command]
Result: PASS / FAIL

### Tests
[command]
Result: PASS / FAIL

## Known Failures
[List failures or write "None."]

## Pre-Existing Failures
[List pre-existing failures tracked separately in KNOWN_ISSUES.md]
```
